# رفع الأداة أونلاين مجانًا (Render)

Render مش زي Firebase Hosting، مبيقبلش رفع ملفات مباشرة — لازم الكود يكون في مستودع GitHub، وهو بيسحبه من هناك ويبنيه تلقائيًا. الخطوات دي مرة واحدة بس.

## 1. حط الكود على GitHub

لو عندك حساب GitHub بالفعل:

```bash
cd website-checker
git init
git add .
git commit -m "أول نسخة من أداة فحص المواقع"
```

بعدين روح على github.com، اعمل مستودع (Repository) جديد فاضي باسم `website-checker`، وربطه بالمشروع:

```bash
git remote add origin https://github.com/USERNAME/website-checker.git
git branch -M main
git push -u origin main
```

(غيّر `USERNAME` باسم حسابك على GitHub)

## 2. اعمل حساب على Render

- روح على render.com
- سجّل حساب مجاني (تقدر تسجل بحساب GitHub بتاعك مباشرة — أسهل)
- **من غير بطاقة ائتمان خالص**

## 3. اربط المستودع

- من لوحة Render، اضغط **New +** ثم **Web Service**
- اختار مستودع `website-checker` اللي رفعته على GitHub
- Render هيكتشف ملف `render.yaml` تلقائيًا ويظبط الإعدادات لوحده (Environment: Docker, Plan: Free)
- لو مكتشفوش لوحده، اختار يدويًا:
  - **Environment**: Docker
  - **Plan**: Free
- اضغط **Create Web Service**

## 4. استنى البناء

أول مرة هياخد وقت أطول شوية (5-10 دقايق) لأن Docker بيحمّل إيميدج فيه متصفح Chromium كامل. بعد ما يخلص، Render هيديك رابط زي:

```
https://website-checker-xxxx.onrender.com
```

افتحه في المتصفح، هتلاقي الأداة شغالة بنفس الشكل اللي شغال بيه على جهازك.

## ملاحظات مهمة عن الخطة المجانية

- **السيرفر بينام بعد 15 دقيقة من عدم الاستخدام**. أول طلب بعد النوم بياخد 30-60 ثانية عشان يصحى، وده طبيعي في الخطة المجانية.
- في حد شهري لساعات التشغيل المجانية، كافي جدًا للاستخدام الشخصي.
- أي تعديل تعمله في الكود وترفعه بـ `git push`، Render بيعيد البناء والنشر تلقائيًا.

## بديل: Railway

نفس الخطوات بالظبط (GitHub → ربط المستودع)، الفرق إن Railway بيدّيك رصيد مجاني شهري بدل "ساعات مجانية غير محدودة"، وممكن يحتاج بطاقة بعد فترة تجريبية حسب سياستهم وقت التسجيل. لو حصل مشكلة في Render جرّب railway.app بنفس ملف الـ Dockerfile من غير أي تعديل.
