# لوحة فحص المواقع

أداة شخصية لاختبار ضغط أي موقع (Load Testing) وعمل فحص شامل للأخطاء (Audit) قبل ما تسلّمه للعميل.

## طريقة التشغيل

1. لازم يكون عندك Node.js نسخة 18 أو أحدث.
2. افتح Terminal داخل مجلد المشروع ونفذ:

```bash
npm install
```

هذا الأمر هيثبت المكتبات المطلوبة، وهيحاول يثبت متصفح Chromium الخاص بـ Playwright تلقائيًا (مطلوب لفحص أخطاء JavaScript). لو حصلت مشكلة في التثبيت التلقائي، نفذ يدويًا:

```bash
npx playwright install chromium
```

3. شغّل السيرفر:

```bash
npm start
```

4. افتح المتصفح على:

```
http://localhost:3000
```

## طريقة الاستخدام

1. حط رابط الموقع اللي عايز تفحصه.
2. أكد إنك تملك الموقع أو عندك إذن لفحصه (شرط أساسي — الأداة مبنية عشان تفحص مواقعك انت، مش مواقع غيرك بدون إذن، لأن اختبار الضغط بدون إذن ممكن يتفسر كهجوم).
3. اضغط "ابدأ اختبار الضغط" لمعرفة الموقع بيستحمل كام زائر متزامن قبل ما يبدأ يتعب.
4. اضغط "ابدأ الفحص الشامل" عشان تكتشف الروابط المكسورة، الصفحات البطيئة، هيدرز الأمان الناقصة، وأخطاء JavaScript.

## ملاحظات مهمة

- **اختبار الضغط بيرفع تدريجيًا** عدد الطلبات المتزامنة (5 → 10 → 25 → 50 → 100 → 200 → 300 → 500) ويوقف تلقائيًا لما يلاقي نسبة فشل أعلى من 30% أو زمن استجابة أعلى من 8 ثواني، ويقولك عند كام زائر حصلت المشكلة.
- **الفحص الشامل** بيزحف على الصفحات الداخلية للموقع (لغاية العدد اللي تحدده)، ويفحص كل صفحة لوحدها.
- الأرقام (أقصى عدد طلبات، أقصى عدد صفحات) محدودة بسقف أمان في الكود (500 طلب / 50 صفحة) عشان الأداة متتحولش لأداة ضرر بالغلط.
- شغّل اختبار الضغط على نسخة تجريبية (staging) من الموقع لو ينفع، مش على النسخة اللي فيها زوار حقيقيين، عشان ميحصلش تعطيل فعلي أثناء الفحص.

## لو حبيت تطورها لمنتج تبيعه للعملاء لاحقًا

البنية الحالية مقسّمة بشكل يسهّل التوسع:

- `server.js` فيه كل منطق الفحص منفصل عن الواجهة — تقدر تحطه وراء نظام تسجيل دخول وحسابات مستخدمين من غير ما تلمس منطق الاختبار نفسه.
- تقدر تضيف قاعدة بيانات تحفظ فيها نتائج كل فحص لكل عميل وتاريخه.
- تقدر تحول الفحوصات لتشتغل في الخلفية (background jobs / queue) بدل ما العميل يستنى في نفس الصفحة، وده هيبقى مهم لما يبقى عندك أكتر من عميل بيشغّل فحص في نفس الوقت.
- السقف الأمني (500 طلب / 50 صفحة) لازم يتراجع ويتظبط حسب خطة كل عميل لو حولتها لمنتج مدفوع.
